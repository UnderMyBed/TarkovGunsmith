**Goal:** Ship the `/sim` route with a three-panel layout, a clickable SVG body silhouette that appends shots via `useScenario`, an editable shot queue (remove + reorder), ammo / helmet / body-armor pickers that render but don't yet feed into a scenario run, and a nav link. **Architecture:** New route `apps/web/src/routes/sim.tsx` composes two new feature components: `BodySilhouette` (SVG, 7 clickable zones) and `ShotQueue` (ordered list with reorder / remove / clear actions). Both dispatch through the `useScenario` hook shipped in PR 2. Pickers reuse the `@tarkov/data` hooks + styling vocabulary from `/calc`. Layout uses Tailwind's responsive grid: 3 columns on `lg:`, stacks below.

**Tech Stack:** React 19, TanStack Router (file-based), Tailwind v4, existing `@tarkov/data` + `@tarkov/ui` primitives. No new runtime deps, no new test deps.

## Scope decisions

1. **No component tests in this PR.** `apps/web` doesn't have `@testing-library/react` or `jsdom`. Components are thin and compose typed primitives. Any extractable pure logic (zone labels, silhouette metadata) is tested as a pure function. Rationale: adding a render harness just for the skeleton is scope creep; component tests follow naturally in PR 4 if the interactive surface grows.
2. **`Run` button is visible but `disabled` in this PR.** Its enabled path requires ammo + target construction, which belongs to PR 4. Showing the button with a tooltip/hint ("Wire-up in progress — see PR 4") is clearer than hiding it.
3. **Pickers render but don't wire into state that matters.** Ammo + helmet + body-armor pickers use local `useState` and display as populated dropdowns. Their selections don't affect the scenario state yet. Distance is a numeric input with local state.
4. **Body silhouette is functional-ugly, not pretty.** Inline SVG with colored rectangles / circle for zones. Keyboard focusable (`<button>` elements wrapping SVG regions). Visual polish is a future follow-up — the spec does not require art.
5. **Mobile layout:** 3 panels stack vertically on `<lg`. Desktop: 3 columns. No custom breakpoints beyond Tailwind's `lg:`.
6. **No accessibility beyond baseline.** Semantic `<button>` / `<select>` / `aria-label`. ARIA live regions or full screen-reader workflow deferred (spec §3 non-goal).

import type { Zone } from "@tarkov/ballistics";
export { ZONES } from "@tarkov/ballistics";

/** Human-readable labels for each zone, used for buttons and `aria-label`. */
export const ZONE_LABELS: Readonly<Record<Zone, string>> = {
  head: "Head",
  thorax: "Thorax",
  stomach: "Stomach",
  leftArm: "Left arm",
  rightArm: "Right arm",
  leftLeg: "Left leg",
  rightLeg: "Right leg",
};

/**
 * Rectangle coordinates for each zone on the 200×320 SVG silhouette. The head
 * uses `x/y` as the circle center and ignores `width`/`height` beyond minima.
 *
 * Origin is top-left; units are SVG user units.
 */
export interface ZoneGeometry {
  readonly x: number;
  readonly y: number;
  readonly width: number;
  readonly height: number;
  /** Shape hint: "rect" or "circle". Defaults to "rect". */
  readonly shape: "rect" | "circle";
}

export const ZONE_GEOMETRY: Readonly<Record<Zone, ZoneGeometry>> = {
  head: { x: 100, y: 30, width: 36, height: 36, shape: "circle" },
  thorax: { x: 75, y: 60, width: 50, height: 60, shape: "rect" },
  stomach: { x: 80, y: 122, width: 40, height: 35, shape: "rect" },
  leftArm: { x: 35, y: 65, width: 30, height: 100, shape: "rect" },
  rightArm: { x: 135, y: 65, width: 30, height: 100, shape: "rect" },
  leftLeg: { x: 72, y: 160, width: 26, height: 130, shape: "rect" },

```tsx
import type { Zone } from "@tarkov/ballistics";
import { ZONES, ZONE_LABELS, ZONE_GEOMETRY } from "./zones.js";

export interface BodySilhouetteProps {
  /** Called when a zone is clicked / keyboard-activated. */
  readonly onZoneClick: (zone: Zone) => void;
}

/**
 * Functional SVG silhouette: a circle (head) + six rectangles (body parts).
 * Each zone is a focusable, accessible `<button>` that calls `onZoneClick`.
 *
 * Visual polish is deliberately minimal in the skeleton — this is a
 * stick-figure hit-area target, not artwork.
 */
export function BodySilhouette({ onZoneClick }: BodySilhouetteProps) {
  return (
    <div className="flex items-center justify-center">
      <svg
        viewBox="0 0 200 320"
        role="group"
        aria-label="Target body silhouette"
        className="h-[320px] w-[200px]"
      >
        {ZONES.map((zone) => {
          const g = ZONE_GEOMETRY[zone];
          const label = ZONE_LABELS[zone];
          const key = zone;
          return (
            <ZoneHitArea
              key={key}
              zone={zone}
              label={label}
              geometry={g}
              onClick={() => onZoneClick(zone)}
            />
          );
        })}
      </svg>
    </div>
  );
}

function ZoneHitArea({
  zone,
  label,
  geometry,
  onClick,
}: {
  zone: Zone;
  label: string;
  geometry: (typeof ZONE_GEOMETRY)[Zone];
  onClick: () => void;
}) {
  const commonClasses =
    "cursor-pointer fill-[var(--color-muted)] stroke-[var(--color-border)] stroke-1 hover:fill-[var(--color-primary)]/30 focus-visible:outline focus-visible:outline-2 focus-visible:outline-[var(--color-primary)]";

  const onKeyDown = (e: React.KeyboardEvent<SVGElement>) => {
    if (e.key === "Enter" || e.key === " ") {
      e.preventDefault();
      onClick();
    }
  };

  if (geometry.shape === "circle") {
    return (
      <circle
        role="button"
        tabIndex={0}
        aria-label={`Add ${label} shot`}
        data-zone={zone}
        cx={geometry.x}
        cy={geometry.y}
        r={geometry.width / 2}
        className={commonClasses}
        onClick={onClick}
        onKeyDown={onKeyDown}
      />
    );
  }
  return (
    <rect
      role="button"
      tabIndex={0}
      aria-label={`Add ${label} shot`}
      data-zone={zone}
      x={geometry.x}
      y={geometry.y}
      width={geometry.width}
      height={geometry.height}
      rx={4}
      className={commonClasses}
      onClick={onClick}
      onKeyDown={onKeyDown}
    />
  );
}
```

```tsx
import type { PlannedShot } from "@tarkov/ballistics";
import { ZONE_LABELS } from "./zones.js";

export interface ShotQueueProps {
  readonly plan: readonly PlannedShot[];
  readonly onMove: (from: number, to: number) => void;
  readonly onRemove: (index: number) => void;
  readonly onClear: () => void;
}

/**
 * Ordered list of planned shots with per-row Remove / ↑ / ↓ buttons and a
 * bulk Clear action. Empty state shows a hint pointing the user at the
 * silhouette.
 */
export function ShotQueue({ plan, onMove, onRemove, onClear }: ShotQueueProps) {
  if (plan.length === 0) {
    return (
      <p className="text-sm text-[var(--color-muted-foreground)]">
        Click a body zone on the silhouette to queue a shot.
      </p>
    );
  }

  return (
    <div className="flex flex-col gap-2">
      <div className="flex items-center justify-between">
        <span className="text-sm font-medium">
          Plan ({plan.length} shot{plan.length === 1 ? "" : "s"})
        </span>
        <button
          type="button"
          onClick={onClear}
          className="text-xs text-[var(--color-muted-foreground)] underline hover:text-[var(--color-destructive)]"
        >
          Clear all
        </button>
      </div>
      <ol className="flex flex-col gap-1">
        {plan.map((shot, i) => (
          <li
            key={i}
            className="flex items-center gap-2 rounded-[var(--radius)] border bg-[var(--color-card)] px-2 py-1.5 text-sm"
          >
            <span className="flex-none text-xs tabular-nums text-[var(--color-muted-foreground)]">
              {i + 1}.
            </span>
            <span className="flex-1 font-medium">{ZONE_LABELS[shot.zone]}</span>
            <span className="flex-none text-xs text-[var(--color-muted-foreground)]">
              {shot.distance}m
            </span>
            <div className="flex flex-none gap-1">
              <button
                type="button"
                aria-label={`Move shot ${i + 1} up`}
                disabled={i === 0}
                onClick={() => onMove(i, i - 1)}
                className="rounded border px-1.5 text-xs disabled:opacity-40"
              >
                ↑
              </button>
              <button
                type="button"
                aria-label={`Move shot ${i + 1} down`}
                disabled={i === plan.length - 1}
                onClick={() => onMove(i, i + 1)}
                className="rounded border px-1.5 text-xs disabled:opacity-40"
              >
                ↓
              </button>
              <button
                type="button"
                aria-label={`Remove shot ${i + 1}`}
                onClick={() => onRemove(i)}
                className="rounded border px-1.5 text-xs text-[var(--color-destructive)]"
              >
                ×
              </button>
            </div>
          </li>
        ))}
      </ol>
    </div>
  );
}
```

```tsx
import { createFileRoute } from "@tanstack/react-router";
import { useMemo, useState } from "react";
import { useAmmoList, useArmorList } from "@tarkov/data";
import { Card, CardContent, CardDescription, CardHeader, CardTitle, Input } from "@tarkov/ui";
import { useScenario } from "../features/sim/useScenario.js";
import { BodySilhouette } from "../features/sim/body-silhouette.js";
import { ShotQueue } from "../features/sim/shot-queue.js";

export const Route = createFileRoute("/sim")({
  component: SimPage,
});

const DEFAULT_DISTANCE = 15;

function SimPage() {
  const ammo = useAmmoList();
  const armor = useArmorList();

  const [ammoId, setAmmoId] = useState<string>("");
  const [helmetId, setHelmetId] = useState<string>("");
  const [bodyArmorId, setBodyArmorId] = useState<string>("");
  const [distance, setDistance] = useState<number>(DEFAULT_DISTANCE);

  const { plan, append, move, remove, clear } = useScenario();

  const ammoOptions = useMemo(
    () => (ammo.data ? [...ammo.data].sort((a, b) => a.name.localeCompare(b.name)) : []),
    [ammo.data],
  );
  const helmetOptions = useMemo(
    () =>
      armor.data
        ? armor.data
            .filter((a) => a.properties.zones.includes("head"))
            .sort((a, b) => a.name.localeCompare(b.name))
        : [],
    [armor.data],
  );
  const bodyArmorOptions = useMemo(
    () =>
      armor.data
        ? armor.data
            .filter(
              (a) => a.properties.zones.includes("chest") || a.properties.zones.includes("thorax"),
            )
            .sort((a, b) => a.name.localeCompare(b.name))
        : [],
    [armor.data],
  );

  const isLoading = ammo.isLoading || armor.isLoading;
  const dataError = ammo.error ?? armor.error;

  return (
    <div className="flex flex-col gap-6">
      <section>
        <h1 className="text-3xl font-bold tracking-tight">Ballistics Simulator</h1>
        <p className="mt-2 text-[var(--color-muted-foreground)]">
          Build a multi-shot engagement — pick ammo, target loadout, then click zones on the
          silhouette to queue shots. Runs through <code>simulateScenario</code> for a shot-by-shot
          outcome. Multi-shot math lands in the next PR.
        </p>
      </section>

      {dataError && (
        <Card>
          <CardContent className="pt-6">
            <p className="text-[var(--color-destructive)]">
              Failed to load data: {dataError.message}
            </p>
          </CardContent>
        </Card>
      )}

      <div className="grid gap-6 lg:grid-cols-3">
        {/* Inputs panel */}
        <Card>
          <CardHeader>
            <CardTitle>Inputs</CardTitle>
            <CardDescription>Ammo + target loadout + distance.</CardDescription>
          </CardHeader>
          <CardContent>
            <form className="flex flex-col gap-3" onSubmit={(e) => e.preventDefault()}>
              <label className="flex flex-col gap-1.5">
                <span className="text-sm font-medium">Ammo</span>
                <select
                  className="h-9 rounded-[var(--radius)] border bg-[var(--color-input)] px-3 text-sm"
                  value={ammoId}
                  onChange={(e) => setAmmoId(e.target.value)}
                  disabled={isLoading || ammoOptions.length === 0}
                >
                  <option value="">{isLoading ? "Loading…" : "Select ammo…"}</option>
                  {ammoOptions.map((a) => (
                    <option key={a.id} value={a.id}>
                      {a.name}
                    </option>
                  ))}
                </select>
              </label>

              <label className="flex flex-col gap-1.5">
                <span className="text-sm font-medium">Helmet (optional)</span>
                <select
                  className="h-9 rounded-[var(--radius)] border bg-[var(--color-input)] px-3 text-sm"
                  value={helmetId}
                  onChange={(e) => setHelmetId(e.target.value)}
                  disabled={isLoading || helmetOptions.length === 0}
                >
                  <option value="">— none —</option>
                  {helmetOptions.map((a) => (
                    <option key={a.id} value={a.id}>
                      {a.name}
                    </option>
                  ))}
                </select>
              </label>

              <label className="flex flex-col gap-1.5">
                <span className="text-sm font-medium">Body armor (optional)</span>
                <select
                  className="h-9 rounded-[var(--radius)] border bg-[var(--color-input)] px-3 text-sm"
                  value={bodyArmorId}
                  onChange={(e) => setBodyArmorId(e.target.value)}
                  disabled={isLoading || bodyArmorOptions.length === 0}
                >
                  <option value="">— none —</option>
                  {bodyArmorOptions.map((a) => (
                    <option key={a.id} value={a.id}>
                      {a.name}
                    </option>
                  ))}
                </select>
              </label>

              <label className="flex flex-col gap-1.5">
                <span className="text-sm font-medium">Distance (m)</span>
                <Input
                  type="number"
                  min={0}
                  max={500}
                  step={1}
                  value={Number.isFinite(distance) ? String(distance) : ""}
                  onChange={(e) => {
                    const next = Number(e.target.value);
                    setDistance(Number.isFinite(next) ? next : 0);
                  }}
                />
              </label>
            </form>
          </CardContent>
        </Card>

        {/* Plan panel */}
        <Card>
          <CardHeader>
            <CardTitle>Shot plan</CardTitle>
            <CardDescription>Click a zone on the silhouette to add a shot.</CardDescription>
          </CardHeader>
          <CardContent className="flex flex-col gap-4">
            <BodySilhouette onZoneClick={(zone) => append({ zone, distance })} />
            <ShotQueue plan={plan} onMove={move} onRemove={remove} onClear={clear} />
            <button
              type="button"
              disabled
              title="Scenario execution lands in Simulator PR 4"
              className="rounded-[var(--radius)] border bg-[var(--color-primary)]/40 px-3 py-1.5 text-sm font-medium text-[var(--color-primary-foreground)] disabled:cursor-not-allowed disabled:opacity-60"
            >
              Run
            </button>
          </CardContent>
        </Card>

        {/* Results panel */}
        <Card>
          <CardHeader>
            <CardTitle>Results</CardTitle>
            <CardDescription>
              Shot-by-shot outcome will appear here once Run is wired up.
            </CardDescription>
          </CardHeader>
          <CardContent>
            <p className="text-sm text-[var(--color-muted-foreground)]">
              No results yet — finish PR 4 to connect the simulator.
            </p>
          </CardContent>
        </Card>
      </div>
    </div>
  );
}
```
